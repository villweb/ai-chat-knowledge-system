# AI 输入策略说明

状态：P0 已确认原则和红线，P2 已按本文实现最小 AI 提炼输入边界（2026-06-23）

本文档定义发送给云端或本地模型前的最小必要输入、遮蔽规则、用户确认内容和日志边界。P0 确认原则和红线，P2 在实现 AI 提炼前再次确认细则。

## 1. 基本原则

- 默认不上传完整原始记录。
- 只发送完成当前提炼任务所需的最小片段。
- 公司项目、客户信息、无法确认权限的数据默认阻断。
- 用户必须在首次 AI 提炼前确认将发送哪些内容。
- 日志不得保存 API Key、完整原文、大段对话或未遮蔽敏感信息。

## 2. 可发送字段

在用户授权后，以下字段可以发送给 AI：

- `record_id`
- `source_app`
- `topic`
- `project`，仅限非敏感项目名
- `message_time`
- `user_message` 的必要摘录
- `ai_message` 的必要摘录
- `context_summary`
- `turn_index`

以下字段默认不发送：

- 本机绝对路径。
- API Key、令牌、密码、环境变量。
- 完整原始文件内容。
- 未经确认的公司、客户或个人敏感信息。
- 日志、命令输出和堆栈信息的大段原文。

`raw_path` 只用于本地追溯，不发送给云端 AI。需要向模型说明来源时，只发送 `source_app` 和脱敏后的来源描述。

## 3. 遮蔽规则

发送前必须遮蔽：

- API Key、访问令牌、密码和 Cookie。
- 本机用户名和绝对路径，替换为 `<local_path>`。
- 邮箱、手机号、身份证号、银行卡号等个人敏感信息，替换为对应占位符。
- 公司客户名、内部项目名、合同信息等业务敏感信息，默认阻断；确需处理时必须单独授权。
- 大段命令输出，只保留必要错误摘要。

遮蔽后的文本仍然无法表达原意时，应阻断本次 AI 提炼，进入人工检查。

## 4. 长度限制

默认限制：

- 单条 `NormalizedRecord` 发送摘录不超过 2000 个中文字符。
- 单次 AI 请求最多处理 10 条 `NormalizedRecord`。
- 单次 AI 请求总输入摘录不超过 12000 个中文字符。
- 单条知识原子的证据摘录不超过 300 个中文字符。

超过限制时，先按主题、时间和用户请求拆分批次，不直接发送完整原文。

## 5. 用户确认界面

首次 AI 提炼前，界面必须展示：

- 将使用的模型服务。
- 将处理的来源。
- 将发送的记录数量。
- 脱敏后的内容预览。
- 不会发送的内容类型。
- 是否包含敏感风险提示。

用户确认后，只代表当前配置和当前来源授权有效。新增来源、切换模型服务或处理敏感来源时，需要重新确认。

## 6. 日志边界

日志允许记录：

- `run_id`
- 来源名称。
- 文件数量。
- 标准化记录数量。
- AI 请求批次数。
- 成功或失败状态。
- 错误类型。

日志不得记录：

- API Key。
- 完整提示词输入。
- 完整原始对话。
- 未遮蔽的本机路径。
- 未遮蔽的敏感信息。

## 7. 验收标准

- 未经用户授权，不能触发 AI 调用。
- 敏感来源默认阻断。
- AI 请求预览中不能出现 API Key、绝对路径或明显敏感信息。
- 日志中不能出现 API Key、完整原文或大段对话。
- 每条知识原子仍能通过本地 `raw_path` 追溯原始记录。
